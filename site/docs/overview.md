# Overview

AI tools can now generate code faster than many teams can review it manually. That creates a reliability and security gap.

Code Review Council addresses this with a multi-agent reviewer panel plus an evidence-based chair that consolidates findings into a single recommendation.

## What Council does

- Runs deterministic checks before LLM analysis.
- Builds a structured review context (ReviewPack) to reduce reviewer guesswork.
- Uses specialist reviewers in parallel, then a chair model to adjudicate findings with evidence.
- Produces outputs for both technical developers and product owners.

## What Council does **not** do

- It does **not** guarantee bug-free or vulnerability-free software.
- It does **not** replace human reviewers.
- It does **not** make universal promises about speed/cost, which depend on model/provider/configuration.
